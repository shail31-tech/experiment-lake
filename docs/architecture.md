# Architecture

ExperimentLake implements the analytics layer of a modern A/B testing platform.

## Data Flow

```text
generate_events.py
  -> send_events.py
  -> API Gateway HTTP API
  -> Lambda ingest function
  -> S3 raw events
  -> Glue raw_events table
  -> Athena CTAS transform
  -> S3 curated Parquet
  -> Python analysis / Streamlit dashboard
```

## AWS Components

- S3 stores raw, processed, curated, and Athena query-result data.
- API Gateway exposes `POST /events` for event ingestion.
- Lambda validates incoming event payloads and writes them to S3.
- Glue Data Catalog describes the raw JSON event table.
- Athena queries raw events and creates curated Parquet tables.
- IAM roles and policies provide scoped access for Lambda and Terraform-managed services.
- CloudWatch Logs receives Lambda execution logs through the basic Lambda execution policy.

## S3 Zones

- Raw: immutable event landing zone.
- Processed: reserved for cleaned intermediate datasets.
- Curated: analytics-ready Parquet tables.
- Athena results: query result files generated by Athena.

## Raw Event Layout

```text
events/
  experiment_id=onboarding_v2/
    event_date=2026-01-01/
      ingest_date=2026-07-18/
        <event_id>.json
```

This partitioned layout lets Athena prune data by experiment and date.

## Curated Table

`user_experiment_facts` contains one row per user per experiment. It deduplicates raw event rows and converts event streams into analysis fields:

- `was_exposed`
- `did_signup`
- `did_activate`
- `did_subscribe`
- `revenue`
- user segments such as device type, country, traffic source, and plan type

This table is the statistical analysis surface for the A/B test.
